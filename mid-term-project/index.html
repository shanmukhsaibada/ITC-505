Project Reflection

Process and Creative Choices

Story Development: The narrative was carefully planned to offer a rich and engaging experience with at least eight different endings. Each path was designed to be unique, providing a sense of discovery and adventure. Characters and plot twists were incorporated to maintain interest and encourage replayability. The choices were structured to impact the storyline significantly, making each decision feel meaningful.

Design and Layout: The visual design aimed to complement the adventure theme. A cohesive color scheme and font selection were used to create an immersive experience. Images were chosen to enhance the storytelling, with a unique image for each ending to provide visual closure to the story paths. The layout was designed to be clean and intuitive, ensuring easy navigation through the story. CSS was used extensively to style the elements, creating a visually appealing and user-friendly interface.

Technical Implementation: The project was built using HTML, CSS, and JavaScript. HTML provided the structure, CSS handled the styling, and JavaScript managed the interactivity. Story stages were represented as objects in JavaScript, each containing the text, choices, consequences, and images. This modular approach made the code easier to manage and extend. Functions were written to start/restart the game, update the page with the current story stage, and handle game endings. Event listeners were used to respond to user choices and progress the story.

Challenges and Solutions

Ensuring Cross-Browser Compatibility: Different browsers may interpret code slightly differently, leading to inconsistencies. Regular testing on multiple browsers (Chrome, Firefox, Safari, Edge) and using polyfills for unsupported features helped achieve consistent behavior across platforms.

Optimizing Performance: Large images and resources can slow down the webpage loading time. Images were compressed and CSS and JavaScript files were minified. This significantly improved loading times and overall performance.

Handling Complex Game Logic: Managing the different story paths and ensuring smooth transitions can be complex. The story was broken down into smaller, manageable objects, with clear relationships between choices and consequences. This modular approach made the code easier to debug and maintain.

Enhancing User Experience: Ensuring the game is intuitive and enjoyable for users. Feedback from a small group of users was gathered and incorporated into the design. Clear instructions and an intuitive layout were prioritized.

Insights Gained

Importance of Planning: Detailed planning of the story flow and structure was crucial. Using diagrams and flowcharts to visualize the story paths helped in organizing the content and ensuring all endings were reachable.

Value of User Feedback: Testing the game with real users provided valuable insights into usability and engagement. User feedback highlighted areas for improvement and helped refine the final product.

Accessibility: Implementing accessibility features not only widened the audience but also improved the overall user experience. Tools like WAVE and Lighthouse were instrumental in identifying and addressing accessibility issues.

Collaboration and Continuous Improvement: Sharing the project with peers and seeking feedback throughout the development process was beneficial. Continuous improvement based on feedback and testing led to a polished final product.
